The project

[Quirijn] The debriefing, what we started with etc

The project

  • A simple web application
  • Preservation of historically valuable data
  • Stored in a repository

[Quirijn] General idea; A web application to import/export the data; Digitally preserve irish historical data; All the data store in repo

What we had to start with

[Quirijn] *Awkward silence*

Nothing

[Quirijn] Nothing; Stared off from scratch with little idea on what it was really about; made lots of mistakes, shown later.

And so we started

  • A single Node.js application
  • Fedora Commons as storage

[Quirijn] Used Fedora as backend;

But then we realised

  • The code wasn't very maintainable
  • Using Fedora Commons as storage wasn't sensible
  • You couldn't easily develop 3rd party applications

[Quirijn]

So we decided to

  • Make a clear differentiation between the front- and back-end
  • Create a developer friendly API
  • Create a front-end using said API
  • Treat Fedora Commons as a blackbox
  • Use MongoDB to store and deliver content for the API
